# Мультимодальные CLI-инструменты OpenAI

Проект предоставляет удобные командные инструменты для работы с мультимодальными моделями OpenAI:
- **TTS** — синтез речи (текст → аудио)
- **STT** — распознавание речи (аудио → текст)
- **Narrate** — по теме: сценарий (в файл) + озвучка
- **Генерация изображений** — создание картинок по текстовому описанию (в т.ч. обучающие на тему)
- **Создание видео** — автоматическая сборка короткого обучающего видео на заданную тему

## Установка

### 1. Виртуальное окружение (рекомендуется)

```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate
```

### 2. Зависимости

```bash
pip install -r requirements.txt
```

### 3. FFmpeg (для создания видео)

MoviePy использует FFmpeg для работы с видео и аудио. Его необходимо установить отдельно.

**Windows** (через winget или chocolatey):
```bash
winget install FFmpeg
# или
choco install ffmpeg
```

**Linux** (Ubuntu/Debian):
```bash
sudo apt update && sudo apt install ffmpeg
```

**macOS**:
```bash
brew install ffmpeg
```

Добавьте FFmpeg в PATH, чтобы система могла его найти.

### 4. ImageMagick (опционально, для текстовых заголовков в видео)

Для наложения текстовых заголовков на кадры видео может потребоваться ImageMagick. Если он не установлен, видео будет собираться без текста поверх изображений.

**Windows**: скачайте с [imagemagick.org](https://imagemagick.org/script/download.php)  
**Linux**: `sudo apt install imagemagick`  
**macOS**: `brew install imagemagick`

### 5. API-ключ

Скопируйте пример конфигурации и укажите свой ключ OpenAI:

```bash
copy .env.example .env
# Отредактируйте .env и вставьте ваш OPENAI_API_KEY
```

## Использование

Все команды запускаются через единую точку входа `main.py`:

### TTS — текст в речь

```bash
python main.py tts "Редомициляция позволяет сменить юрисдикцию компании" speech.mp3
```

С дополнительными параметрами:
```bash
python main.py tts "Текст для озвучки" output.mp3 --voice nova --model gpt-4o-mini-tts
```

### STT — речь в текст

```bash
python main.py stt speech.mp3 transcript.txt
```

```bash
python main.py stt input.wav output.txt --model gpt-4o-mini-transcribe
```

### Narrate — тема → сценарий → озвучка

По заданной теме генерируется текстовый сценарий (сохраняется в файл), затем он озвучивается. Удобно для подкастов и образовательных аудиороликов.

**Общая форма:**
```bash
python main.py narrate --topic "Ваша тема" [--output аудиофайл] [--script-output файл_сценария] [--voice голос]
```

**Пример по редомициляции:**
```bash
python main.py narrate --topic "Редомициляция компании" --output озвучка.mp3 --script-output сценарий.txt --voice nova
```

Результат: файл `сценарий.txt` с текстом и файл `озвучка.mp3` с озвучкой. Дополнительно: `--model gpt-4o` (GPT для сценария), `--tts-model gpt-4o-mini-tts` (модель TTS).

### Генерация изображений и обучающие картинки на тему

Команда `img` создаёт картинку по **текстовому промпту**. Тему задаёте вы: что именно нарисовать и в каком стиле.

**Общая форма:**
```bash
python main.py img "описание картинки" выходной_файл [--size 1024x1024] [--model gpt-image-1]
```

**Что указать в промпте для обучающей картинки:**
- Тему (о чём инфографика/схема).
- Стиль: например «инфографика», «flat design», «схема».
- Цвета и формат: «16:9», «синий и зелёный», «светлый фон».
- Элементы: стрелки, подписи, иконки, пиктограммы (часы, щит, деньги и т.д.).

**Пример — одна обучающая картинка по редомициляции (альбомный формат, без обрезки):**
```bash
python main.py img "Инфографика 16:9 о процессе редомициляции компании. Flat design, синий и зелёный, светлый фон. В центре иконка компании. Влево стрелка «Старая юрисдикция», вправо «Новая юрисдикция». Этапы: Решение акционеров, Документы, Регистрация, Исключение из реестра. Пиктограммы: часы (сроки), щит (защита), деньги (налоги). Подпись внизу: Редомициляция — переезд бизнеса без закрытия." редомициляция.png --size 1536x1024
```
Для инфографики и 16:9 используйте альбомный размер `1536x1024` (gpt-image-1), чтобы картинка не была обрезана.

**Другой пример — картинка на свою тему:**
```bash
python main.py img "Инфографика по теме налогообложение для самозанятых: flat design, синий и зелёный, блоки и стрелки, ключевые цифры и этапы" налоги.png
```

Доступные модели: `gpt-image-1` (по умолчанию), `dall-e-3`. Размеры: для инфографики/16:9 — **`1536x1024`** (альбом, без обрезки); квадрат `1024x1024`; для DALL-E 3 — `1792x1024`.

### Создание обучающего видео

Полная генерация видео на тему «редомициляция» (сценарий создаётся автоматически через GPT):

```bash
python main.py video --topic "редомициляция" --output explainer.mp4 --voice nova
```

С использованием готового сценария (JSON):

```bash
python main.py video --script-file script.json --output explainer.mp4
```

Сохранить временные файлы (изображения, аудио) после сборки:

```bash
python main.py video --topic "налогообложение" --output video.mp4 --keep-temp
```

#### Формат JSON-сценария

```json
[
  {
    "part_text": "Текст первой части повествования...",
    "image_prompt": "Detailed description of the image in English..."
  },
  {
    "part_text": "Текст второй части...",
    "image_prompt": "Description for second image..."
  },
  {
    "part_text": "Текст третьей части...",
    "image_prompt": "Description for third image..."
  }
]
```

## Промпты для каждого запроса

Все промпты вынесены в `prompts.py`. Ниже — полные тексты.

### 1. Промпт для озвучки текста (сценарий TTS)

Используется при генерации сценария через `video --topic`. GPT создаёт текст, который затем озвучивается.

```
Напиши сценарий для озвучки образовательного видео на тему «{topic}» продолжительностью 1,5–2 минуты.

Сценарий должен включать:
- Вступление: краткое объяснение сути темы (что это, зачем нужно).
- Основные аспекты: ключевые причины, мотивы, предпосылки.
- Этапы/процесс: как это делается, какие шаги (с примерными сроками, если применимо).
- Плюсы: выгоды, преимущества, сохранение важных атрибутов.
- Риски и сложности: о чём важно знать, возможные подводные камни.
- Заключение: краткий вывод и рекомендация обратиться к специалистам.

Язык — простой, без канцеляризмов, с примерами из жизни. Тон — дружелюбный, уверенный. Текст должен легко читаться диктором.
```

**Пример для темы «Редомициляция компании»:**
```
Напиши сценарий для озвучки образовательного видео на тему «Редомициляция компании: что это и зачем нужно?» продолжительностью 1,5–2 минуты.
Сценарий должен включать: вступление (что такое редомициляция), причины (налоги, активы, рынки), этапы (2–6 месяцев), плюсы (история, контракты), риски (законодательство, налоги), заключение (консультация специалистов).
Язык — простой, без канцеляризмов. Тон — дружелюбный, уверенный.
```

### 2. Промпт для генерации изображения (инфографика)

Для CLI: `python main.py img "ваш промпт" image.png`. При видео — `image_prompt` из сценария.

```
Создай инфографику в формате 16:9, иллюстрирующую тему: {description}.
Плоский стиль (flat design), цвета — синий и зелёный, фон светлый.
Центральный образ, стрелки, блоки, подписи, пиктограммы (часы, щит, деньги).
Крупная подпись внизу. Без мелких деталей, читаемо на экране.
```

**Пример для редомициляции:**
```
Создай инфографику 16:9 о процессе редомициляции. Flat design, синий/зелёный. В центре — иконка компании. Влево стрелка «Старая юрисдикция», вправо «Новая юрисдикция». Этапы над стрелками. Пиктограммы: часы, щит, деньги. Подпись: «Редомициляция — переезд бизнеса без закрытия».
```

### 3. Промпт для видео (структура сцен)

Задаёт GPT разбивку на 6 сцен с `part_text` и `image_prompt` для каждой.

```
Создай сценарий для обучающего видеоролика на тему «{topic}» (~90 сек).
Верни JSON-массив из шести объектов с полями part_text (русский, для озвучки) и image_prompt (английский, 16:9, flat design).
Сцены: 1) титул, 2) суть темы, 3) причины, 4) этапы, 5) плюсы и риски, 6) заключение.
Только JSON.
```

### 4. Текст в речь (TTS) и 5. Речь в текст (STT)

Промптов нет: TTS получает текст (`input=text`), STT — аудиофайл (`file=audio_file`).

## Структура проекта

```
.
├── .env               # Переменные окружения (не коммитить!)
├── .env.example       # Пример конфигурации с ключами

├── temp/              # Временные файлы (создаётся автоматически)
└── README.md
```
